# Admin Screen Reference

## 1. Dashboard summary
Purpose:
High-level BOCRA overview.

Sections:
- summary cards
- escalated complaint counts
- investigation counts
- operator risk summary
- trend widgets

## 2. Escalated complaints queue
Purpose:
Primary operational queue for BOCRA staff.

Columns:
- reference number
- operator
- category
- status
- submitted date
- escalated date

## 3. Complaint detail
Purpose:
Investigation workspace.

Sections:
- complaint summary
- operator snapshot
- documents
- message timeline
- escalation policy snapshot
- regulatory actions

## 4. Operator oversight
Purpose:
Compare regulated operators.

Fields:
- name
- compliance score
- risk level
- license status
- complaint count
- escalation count

## 5. Licensing overview
Purpose:
Show complaint-driven licensing oversight.

Sections:
- active / under review / suspended counts
- high-risk operators
- recommended action indicators
